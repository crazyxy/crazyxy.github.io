LeetCode 398 - Random Pick Index

Posted on 2016-09-16 |

Solution

概率题。每次pick时遍历数组，记录已经访问到的target的个数cnt（包括当前的元素）。如果当前访问的数等于target，则该下标有1/cnt的概率被选中，证明如下：

  • 归纳法。
    • 数组中只有一个元素，被选中的概率是1；
    • 数组中有两个元素，第一个元素被选中的概率是1，访问到第二个元素的时候，该元素被选中的概率是1/2，而第一个元素被选中的概率是1*1/2；
    • 假设当数组中有k个元素，被选中的元素的概率是1/k；
    • 当遇到第k+1个元素时，第k+1个元素被选中的概率为1/(k+1)，而之前每个元素被选中的概率都是(1-1/(k+1))*1/k=1/(k+1);

LeetCode 126 - Word Ladder II

Posted on 2016-09-15 |

Solution

有点难。看到题目的第一反应就是BFS，一开始是想在队列中把保存当前访问记录的vector，数据结构长这样queue<vector<string>>，为了判断是否已经访问过该字符串，在遍历节点的时候建立一个set，结果显然TLE。

一种正确的解法是用BFS层次遍历，直至访问到目的字符串，在遍历的过程中记录每个访问节点的父亲节点。再利用DFS构造路径。

LeetCode 391 - Perfect Rectangle

Posted on 2016-09-14 |

Solution

首先根据给出的数组算出最小和最大的x与y，分别计为minx，maxx，miny，maxy，恰好完全覆盖的必要条件之一就是小矩形的面积和等于(maxx-minx)*(maxy-miny)。其次满足了第一个条件并不一定是完全覆盖，可能有两个矩形有重合，因此还需要其他的条件。
通过对矩形顶点的分类辅助判断。一个完全覆盖的矩形内部的点有三种状态，分别是：只有一个矩形以之为顶角；两个矩形以之为顶角；四个矩形以之为顶角。所以利用set将出现超过一次的顶点都去除，剩下的就是四个顶点。

LeetCode 44 - Wildcard Matching

Posted on 2016-09-13 |

Solution 1

动态规划。dp[i][j]=(dp[i-1][j-1] && (s[i-1]==p[j-1]||p[j-1]=='?')) || ((dp[i-1][j] || dp[i][j-1]) && (p[j-1] == '*'))。时间复杂度O(n*m)

Code 1

Solution 2

设置两个指针i，j，如果s[i] == p[j] || p[j] == '?'则i++,j++；如果p[j] == '*'，记录下该星号的位置，并且记录i开始匹配的位置ss；如果p[j]为其他字符，则查看在j之前是否有*，如果没有，则说明匹配失败；否则将i开始匹配的位置ss加1，j从star+1开始匹配。

LeetCode 4 - Median of Two Sorted Arrays

Posted on 2016-09-12 |

Solution

搜索第K大的数，两个数组同时二分搜索。从数组长度短的数组中取第K/2大，从较长的数组中取K-K/2。如果数组长度len不够则一个取len，另外一个取K-len。如果对应的数x与y相等，则第K大的数就是x。否则根据x与y的大小关系从两个数组相应部分进行递归搜索。
